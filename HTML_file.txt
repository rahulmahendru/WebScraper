        <p>
         One of the most important ways to develop your skills and improve your employability as a software engineer is to work on side projects. These can run the gamut from open source projects to helpful tools and scripts, to fun toys.
        </p>
        <p>
         While some of our colleagues always seem to be working on a new side project, it’s not so easy for the rest of us. We
         <em>
          want
         </em>
         to work on a side project but struggle to come up with a good idea.
        </p>
        <p>
         In my experience, this happens when we  subconsciously artificially restrict ourselves. For example, we may unintentionally limit the types of projects that we feel are ‘worthy’ as side projects.
        </p>
        <p>
         By setting the bar too high — like aiming to create a successful, popular, and original open source library — we rob ourselves of the opportunity to have fun and learn something from a less ambitious, but still worthwhile, side project.
        </p>
        <p>
         In this post, I want to help fight “side project paralysis” by offering up 40 side project ideas that any software developer can start in one weekend. The emphasis here is not on projects that are going to change the world. Instead, these are fun projects to give you something to add to your portfolio
         <strong>
          and
         </strong>
         make you a better developer.
        </p>
        <h2 id="computer-science-side-projects">
         <a aria-hidden="true" class="header-anchor" href="#computer-science-side-projects">
          <i aria-hidden="true" class="fa fa-link">
          </i>
         </a>
         <strong>
          Computer Science Side Projects
         </strong>
        </h2>
        <p>
         <em>
          These projects are suited for engineers trying to beef up their computer science fundamentals. They’ll help you learn more about specific aspects of computing, like file transfer and operating systems design. These projects can be particularly useful for self-taught developers who’ve focused more on practical skills than comp sci fundamentals.
         </em>
        </p>
        <ol>
         <li>
          <p>
           <strong>
            FTP Client
           </strong>
           (suggested implementation: web, desktop or CLI app).
           <br/>
           Build a simple FTP client. As a bonus challenge, support secure file transfer.
          </p>
         </li>
         <li>
          <p>
           <strong>
            HTTP Server
           </strong>
           (suggested implementation: any programming language you want to master). To understand HTTP deeply, build an HTTP server. There are a number of
           <a href="https://ruslanspivak.com/lsbaws-part1/" rel="nofollow noopener" target="_blank">
            tutorials
           </a>
           available to guide you through the process.
          </p>
         </li>
         <li>
          <p>
           <strong>
            Build a simple operating system
           </strong>
           (suggested implementation: any programming language you want to master). This is a project that is technically challenging and will deepen your understanding of how computers and operating systems work. You might wish to start with the free (and cleverly named) book,
           <a href="https://tuhdo.github.io/os01/" rel="nofollow noopener" target="_blank">
            Operating Systems: From 0 to 1
           </a>
           .
          </p>
         </li>
         <li>
          <p>
           <strong>
            Build a web scraper
           </strong>
           (suggested implementation: any programming language you want to master). Build a tool that takes a URL as input and returns the content of the URL as HTML or XML.
          </p>
         </li>
         <li>
          <p>
           <strong>
            Bandwidth monitor
           </strong>
           (suggested implementation: live updating CLI tool). Build a tool to track how much data you have downloaded or uploaded on the internet. Have it email you a weekly report of your usage. As a bonus challenge, predict peak usage times.
          </p>
         </li>
         <li>
          <p>
           <strong>
            Implement a cipher
           </strong>
           (suggested implementation: any programming language you want to master). Build a tool that takes a string of text as input and encrypts it using a cipher, such as the
           <a href="https://en.wikipedia.org/wiki/Caesar_cipher" rel="nofollow noopener" target="_blank">
            Caesar cipher
           </a>
           .
          </p>
         </li>
         <li>
          <p>
           <strong>
            Create a text message downtime alerter
           </strong>
           (suggested implementation: app hooked up to the
           <a href="https://www.twilio.com/docs/api/rest" rel="nofollow noopener" target="_blank">
            Twilio API
           </a>
           for sending text messages). Create an app that takes a URL as input and sends you a text message when a request to the URL returns a response code other than 200 (OK).
          </p>
         </li>
        </ol>
        <h2 id="personal-finance-side-projects">
         <a aria-hidden="true" class="header-anchor" href="#personal-finance-side-projects">
          <i aria-hidden="true" class="fa fa-link">
          </i>
         </a>
         <strong>
          Personal Finance Side Projects
         </strong>
        </h2>
        <p>
         <em>
          These projects will help you achieve a practical goal (get a better handle on your finances), while also improving your software engineering skills.
         </em>
        </p>
        <ol start="8">
         <li>
          <p>
           <strong>
            A net worth calculator and tracker
           </strong>
           (suggested implementation: CLI, web, or mobile app). Build a calculator you can use to track the rise or fall of your net worth on a monthly basis. You can use something like this
           <a href="http://www.schwabmoneywise.com/public/file/P-4038856/Net-Worth-Worksheet.pdf" rel="nofollow noopener" target="_blank">
            net worth worksheet
           </a>
           from Charles Schwab to guide you. Optional extension: have it send you a 12 month report for the previous year on the first of January each year.
          </p>
         </li>
         <li>
          <p>
           <strong>
            A tax forecaster
           </strong>
           (suggested implementation: web app). This will be particularly useful if you do any freelance software engineering. Build a tool that takes your freelance earnings as input and then predicts your expected tax liability for the rest of the financial year. Make it smart enough to predict periods of higher or lower demand for your services, and adjust accordingly.
          </p>
         </li>
         <li>
          <p>
           <strong>
            A deal finder
           </strong>
           (suggested implementation: web app with mobile notifications). Build a simple web app to notify you when an item you covet goes on sale for a good price. You could use a web scraper to pull the item’s product page and notify you of any price changes.
          </p>
         </li>
         <li>
          <p>
           <strong>
            An expense tracker
           </strong>
           (suggested implementation: web or mobile app). Create a simple interface you can use to add and categorize your expenses. Generate monthly reports based on the inputs and write custom alerts for things, like, “spending too much money on coffee… as always.”
          </p>
         </li>
         <li>
          <p>
           <strong>
            A financial independence calculator
           </strong>
           (suggested implementation: web app).
           <a href="https://www.reddit.com/r/financialindependence/" rel="nofollow noopener" target="_blank">
            Financial independence
           </a>
           is, essentially, saving and investing as much of your income as possible so that you don’t
           <em>
            need
           </em>
           to work for money. While many financially independent people continue to work, they can now focus on doing work that they love, rather than work that pays the most. Build a tool to calculate, based on your: current savings, investments, income, retirement accounts, and expenses, how far away you are from financial independence. Some examples:
           <a href="https://www.firecalc.com/" rel="nofollow noopener" target="_blank">
            FIREcalc
           </a>
           ,
           <a href="http://www.cfiresim.com/" rel="nofollow noopener" target="_blank">
            cFIREsim
           </a>
           .
          </p>
         </li>
         <li>
          <p>
           <strong>
            A bill splitter
           </strong>
           (suggested implementation: mobile app). Build a simple tool to help you and your friends split bills when you go out to eat together.
          </p>
         </li>
        </ol>
        <h2 id="games-and-simulation-side-projects">
         <a aria-hidden="true" class="header-anchor" href="#games-and-simulation-side-projects">
          <i aria-hidden="true" class="fa fa-link">
          </i>
         </a>
         <strong>
          Games and Simulation Side Projects
         </strong>
        </h2>
        <p>
         <em>
          Most software engineers I know are fascinated by the world of game development, graphics, and simulations, but don’t have a lot of experience with them. These projects are small enough that you can set foot into this world without biting off more than you can chew.
         </em>
        </p>
        <ol start="14">
         <li>
          <p>
           <strong>
            A random name generator
           </strong>
           (suggested implementation: CLI, web, or mobile app). Build a random name generator (
           <a href="https://www.behindthename.com/random/" rel="nofollow noopener" target="_blank">
            example
           </a>
           ) that creates unique names on the fly, based on an algorithm. Use machine learning techniques to help you by training the program with a sample data set of names similar to those you want to generate. Otherwise, create your own lexical rules for how names are generated. For example, a name generating algorithm inspired by
           <a href="https://en.wikipedia.org/wiki/The_Handmaid%27s_Tale" rel="nofollow noopener" target="_blank">
            The Handmaid's Tale
           </a>
           might stipulate that names for Handmaid women should start with 'Of', and end with a random male name, e.g., Ofpeter.
          </p>
         </li>
         <li>
          <p>
           <strong>
            <a href="https://en.wikipedia.org/wiki/Conway%27s_Game_of_Life" rel="nofollow noopener" target="_blank">
             Conway’s Game of Life
            </a>
           </strong>
           (suggested implementation: any platform capable of real-time graphical rendering). Conway’s Game of Life simulates the lives of simple cells that obey algorithmic laws. This video explains how the game works and includes an example of one possible result:
          </p>
         </li>
        </ol>
        <p>
         <a class="cm-embed" data-card-controls="0" href="https://www.youtube.com/watch?v=C2vgICfQawE&amp;feature=player_embedded">
          epic conway's game of life
         </a>
        </p>
        <ol start="16">
         <li>
          <p>
           <strong>
            A procedurally generated map maker
           </strong>
           (suggested implementation: browser-based app). Create a browser-based application that allows users to procedurally generate a terrain map based on a random seed. The map can be as detailed or as simple as you’d like. This project is a good opportunity to learn about
           <a href="https://en.wikipedia.org/wiki/Procedural_generation" rel="nofollow noopener" target="_blank">
            procedural generation
           </a>
           .
          </p>
         </li>
         <li>
          <p>
           <strong>
            A character generator
           </strong>
           (suggested implementation: browser-based or mobile app, CLI). Create a tool that allows you to randomly generate playable characters for your favorite role-playing games, whether they be tabletop games, like Pathfinder, or video games, like Divinity: Original Sin.
          </p>
         </li>
         <li>
          <p>
           <strong>
            Interactive fiction
           </strong>
           (suggested implementation: CLI). A fun way to get into game development without having to worry about graphical assets, interactive fiction renders the world for the player through text descriptions.
           <a href="http://eblong.com/zarf/zweb/dreamhold/" rel="nofollow noopener" target="_blank">
            The Dreamhold
           </a>
           is a good example of interactive fiction with a useful ‘help’ command.
          </p>
         </li>
        </ol>
        <h2 id="artificial-intelligence-side-projects">
         <a aria-hidden="true" class="header-anchor" href="#artificial-intelligence-side-projects">
          <i aria-hidden="true" class="fa fa-link">
          </i>
         </a>
         <strong>
          Artificial Intelligence Side Projects
         </strong>
        </h2>
        <p>
         <em>
          AI’s usefulness in day-to-day software engineering is increasing by leaps and bounds. It’s now easier than ever before to make your first forays into the world of Artificial Intelligence.
         </em>
        </p>
        <ol start="19">
         <li>
          <p>
           <strong>
            An unbeatable Tic-Tac-Toe engine
           </strong>
           (suggested implementation: CLI program). For an excellent first AI project, try to write an engine that cannot be beaten at Tic-Tac-Toe. You can achieve this by implementing this
           <a href="https://en.wikipedia.org/wiki/Tic-tac-toe#Strategy" rel="nofollow noopener" target="_blank">
            strategy
           </a>
           , which produces a draw as its worst-case outcome.
          </p>
         </li>
         <li>
          <p>
           <strong>
            A chess engine
           </strong>
           (suggested implementation: an engine written in a programming language you want to learn, or master). Try to write an engine that can play chess against a human opponent using a
           <a href="http://wbec-ridderkerk.nl/html/UCIProtocol.html" rel="nofollow noopener" target="_blank">
            Universal Chess Interface
           </a>
           compatible GUI, such as XBoard. See
           <a href="https://github.com/official-stockfish/Stockfish" rel="nofollow noopener" target="_blank">
            Stockfish
           </a>
           as an example. For a less daunting challenge, you may wish to focus on the behavior of just one piece, e.g., Knights.
          </p>
         </li>
         <li>
          <p>
           <strong>
            A niche chatbot
           </strong>
           (suggested implementation: web app). Some of the greatest minds in the world are working on chatbots that respond in lifelike ways. It’s an incredibly difficult challenge, but, by reducing the scope of your chatbot, you have a side project that is more approachable for evenings and weekends. Create a chatbot that produces real-sounding responses based on a niche topic that you’re passionate about: your favorite band, video game, sports team, or TV show. Leverage an existing library to help you, such as
           <a href="https://github.com/gunthercox/ChatterBot" rel="nofollow noopener" target="_blank">
            ChatterBot
           </a>
           .
          </p>
         </li>
         <li>
          <p>
           <strong>
            A spam classifier
           </strong>
           (suggested implementation: any programming language you want to master). Build a tool to classify whether an email is or isn’t spam based on the content alone. You can use this
           <a href="https://www.kaggle.com/wcukierski/enron-email-dataset" rel="nofollow noopener" target="_blank">
            public data set of emails from the Enron investigation
           </a>
           to test your spam classifier.
          </p>
         </li>
        </ol>
        <h2 id="entertainment-side-projects">
         <a aria-hidden="true" class="header-anchor" href="#entertainment-side-projects">
          <i aria-hidden="true" class="fa fa-link">
          </i>
         </a>
         <strong>
          Entertainment Side Projects
         </strong>
        </h2>
        <p>
         <em>
          These projects are for software engineers who want to work on something fun and light-hearted.
         </em>
        </p>
        <ol start="23">
         <li>
          <p>
           <strong>
            A movie showtime finder
           </strong>
           (suggested implementation: web or mobile app with email or text message notifications). Build a program that notifies you, by text or email, about showtimes for potentially interesting movies playing at your favorite cinema. The concept of an 'interesting movie' can be derived using machine learning (if you watch enough movies to have good training data), or a handcrafted algorithm. For example, you might use the
           <a href="http://www.omdbapi.com/" rel="nofollow noopener" target="_blank">
            Open Movie Database API
           </a>
           , paired with an HTML parser like
           <a href="https://www.crummy.com/software/BeautifulSoup/bs4/doc/" rel="nofollow noopener" target="_blank">
            BeautifulSoup
           </a>
           , to build a program that alerts you to sci-fi movies rated 7.0 or above on IMDB, movies starring Amy Adams, and/or any movie with an average rating of 8.0 or higher.
          </p>
         </li>
         <li>
          <p>
           <strong>
            A spoiler blocker
           </strong>
           (suggested implementation: browser extension). Short of implementing a total social media/internet/watercooler chat ban, it can be difficult to avoid seeing spoilers for your favorite TV show… especially if that TV show is as popular as Game of Thrones. Create a browser extension that removes all mentions of your favorite show from any webpage loaded in your browser… or replaces them with cute pictures of puppies.
          </p>
         </li>
        </ol>
        <h2 id="fun-side-projects">
         <a aria-hidden="true" class="header-anchor" href="#fun-side-projects">
          <i aria-hidden="true" class="fa fa-link">
          </i>
         </a>
         <strong>
          Fun Side Projects
         </strong>
        </h2>
        <p>
         <em>
          These projects are all different, from logging sensor data to finding new desktop backgrounds for your computer. One thing they all have in common is that they’re fun, and several of them will have you work with interesting APIs.
         </em>
        </p>
        <ol start="25">
         <li>
          <p>
           <strong>
            Pixel art generator
           </strong>
           (suggested implementation: any programming language you want to master). Build a tool that takes an image as input and samples the image to produce pixel art as output. If you want to improve your front-end skills, generate the resulting pixel art using CSS.
          </p>
         </li>
         <li>
          <p>
           <strong>
            Music suggestion tool
           </strong>
           (suggested implementation: build a wrapper for the
           <a href="https://developer.spotify.com/web-api/get-the-users-currently-playing-track/" rel="nofollow noopener" target="_blank">
            Spotify API
           </a>
           ). Create a tool that tracks the music you listen to and generates a playlist with similar qualities, but of songs you haven’t heard before. The Spotify API provides all of the tools needed to extract what you’ve listened to and create a playlist — the recommendation engine is up to you!
          </p>
         </li>
         <li>
          <p>
           <strong>
            Temperature logger
           </strong>
           (suggested implementation: Raspberry Pi, temperature sensor, web app). Have your Raspberry Pi hooked up to a temperature sensor and send temperature data from your home to a web app that saves the data to a database or updates a CSV file. For bonus points, have a weekly temperature report emailed to you. Other Raspberry Pi projects:
           <a href="https://electronicsofthings.com/videos/home-automation-hand-gesture/" rel="nofollow noopener" target="_blank">
            home automation
           </a>
           ,
           <a href="https://electronicsforu.com/electronics-projects/iot-video-surveillance-raspberry-pi" rel="nofollow noopener" target="_blank">
            home security
           </a>
           .
          </p>
         </li>
         <li>
          <p>
           <strong>
            Microlearning app
           </strong>
           (suggested implementation: web or mobile app). Build an application that sends you one page per day about something you want to learn. This could be a random page from Wikipedia, a page of
           <a href="https://reactjs.org/docs/hello-world.html" rel="nofollow noopener" target="_blank">
            React
           </a>
           documentation, a
           <a href="https://jisho.org/" rel="nofollow noopener" target="_blank">
            Kanji
           </a>
           character, or a page from the
           <a href="https://www.cia.gov/library/publications/the-world-factbook/" rel="nofollow noopener" target="_blank">
            CIA World Factbook
           </a>
           .
          </p>
         </li>
         <li>
          <p>
           <strong>
            Slack bot
           </strong>
           (suggested implementation:
           <a href="https://api.slack.com/" rel="nofollow noopener" target="_blank">
            Slack API
           </a>
           ). If you or your team use the popular chat app Slack, build a bot to make some aspect of your (or your team’s) life easier. Ideas: a coffee order bot, a bot that reports daily on the number of commits made to your team repos, or a daily stand-up reminder bot.
          </p>
         </li>
         <li>
          <p>
           <strong>
            Daily desktop background
           </strong>
           (suggested implementation:
           <a href="https://unsplash.com/developers" rel="nofollow noopener" target="_blank">
            Unsplash API
           </a>
           , scripting language for your OS). Build an app that refreshes your desktop background with a new image every day.
          </p>
         </li>
        </ol>
        <h2 id="coding-challenge-side-projects">
         <a aria-hidden="true" class="header-anchor" href="#coding-challenge-side-projects">
          <i aria-hidden="true" class="fa fa-link">
          </i>
         </a>
         <strong>
          Coding Challenge Side Projects
         </strong>
        </h2>
        <p>
         <em>
          Rather than building a project from scratch, these are collections of coding challenges to improve your skills. If you want to get better at solving discrete problems, or improving your chops with a new programming language, these are all good options.
         </em>
        </p>
        <ol start="31">
         <li>
          <p>
           <strong>
            <a href="http://adventofcode.com/" rel="nofollow noopener" target="_blank">
             Advent of Code
            </a>
           </strong>
           (suggested implementation: any language you want to practice). Advent of Code challenges occur every year in the days leading up to Christmas. They include lots of fun programming challenges that can help you develop your chops in a programming language you’re learning or one you want to master. The challenges from
           <a href="https://adventofcode.com/2015" rel="nofollow noopener" target="_blank">
            2015
           </a>
           ,
           <a href="https://adventofcode.com/2016" rel="nofollow noopener" target="_blank">
            2016
           </a>
           , and
           <a href="https://adventofcode.com/2017" rel="nofollow noopener" target="_blank">
            2017
           </a>
           are still available.
          </p>
         </li>
         <li>
          <p>
           <strong>
            <a href="https://www.kaggle.com/c/titanic" rel="nofollow noopener" target="_blank">
             Kaggle’s Titanic Challenge
            </a>
           </strong>
           (suggested implementation: Python or R). Kaggle offers a fascinating challenge to introduce you to the basics of machine learning with Python or R: use a real data set from the Titanic passenger log to predict which passengers were most likely to survive the disaster.
          </p>
         </li>
         <li>
          <p>
           <strong>
            <a href="https://projecteuler.net/archives" rel="nofollow noopener" target="_blank">
             Project Euler
            </a>
           </strong>
           (suggested implementation: any programming language you want to practice). Project Euler challenges are dedicated to exploring the marriage between mathematics and programming. Use programming tools to solve various mathematical challenges.
          </p>
         </li>
        </ol>
        <h2 id="daily-life-side-projects">
         <a aria-hidden="true" class="header-anchor" href="#daily-life-side-projects">
          <i aria-hidden="true" class="fa fa-link">
          </i>
         </a>
         <strong>
          Daily Life Side Projects
         </strong>
        </h2>
        <p>
         <em>
          These side projects have a practical purpose: they help with some aspect of your daily life. Though they solve simple real-world problems (things like what to have for lunch), the solutions can be as simple, or as complex, as you’d like.
         </em>
        </p>
        <ol start="34">
         <li>
          <p>
           <strong>
            Lunch picker
           </strong>
           (suggested implementation: CLI, web or mobile app). If you’re a working software engineer, you’ve probably wrestled with one of the toughest questions in software development… where should I have lunch? Your lunch picker is the tool you’ll turn to to answer this question. It can be super simple and pick from a range of options you know you like at random, or more complex — pulling in data from Google Reviews and taking into account: location, price, and type of cuisine.
          </p>
         </li>
         <li>
          <p>
           <strong>
            Date planner
           </strong>
           (suggested implementation: mobile app). One of the hardest things about dating, or being in a long-term relationship, is deciding where to go on dates. Build a tool that scours restaurant reviews, event calendars, and other data for date idea suggestions.
          </p>
         </li>
         <li>
          <p>
           <strong>
            Tool to simplify your email newsletters
           </strong>
           (suggested implementation: scripts hooked up to a mail server). If you’re like me, you receive way too many email newsletters every week and they clutter up your inbox. Instead of unsubscribing from all of them in a fit of rage, build a tool that captures these emails and compiles them all into a single email that is sent once a week.
          </p>
         </li>
         <li>
          <p>
           <strong>
            ‘Bring your umbrella’ notifier
           </strong>
           (suggested implementation: mobile app). Build a simple mobile app that sends you a phone notification in the morning telling you if you should bring your umbrella to work (e.g., if it is going to rain in your area that day).
          </p>
         </li>
         <li>
          <p>
           <strong>
            Collection tracker
           </strong>
           (suggested implementation: web or mobile app). Build a tool to keep track of something you collect. You could keep track of: the value of the item, the year it was made, its condition, and its location.
          </p>
         </li>
        </ol>
        <h2 id="data-side-projects">
         <a aria-hidden="true" class="header-anchor" href="#data-side-projects">
          <i aria-hidden="true" class="fa fa-link">
          </i>
         </a>
         <strong>
          Data Side Projects
         </strong>
        </h2>
        <p>
         <em>
          These projects will improve your ability to work with data. Software engineers work with ever-larger amounts of data. These projects will help you learn how to collate, parse, and analyze data with confidence.
         </em>
        </p>
        <ol start="39">
         <li>
          <p>
           <strong>
            Create and automatically update a data set
           </strong>
           (suggested implementation: web scraper input to CSV output). Build a tool to automatically build and update a data set about something you’re interested in. Suggested data sets: stats about your favorite sports team, flights to destinations you want to visit, meteorological data from where you live, or anything else that interests you. The most important thing is that the data set should require constant updates as new data is generated — and should occur automatically. For example, as soon as your sport’s team’s results are posted on a website, the data should be automatically scraped and added to your data set.
          </p>
         </li>
         <li>
          <p>
           <strong>
            Enter a data science competition
           </strong>
           (suggested implementation: R or Python). Kaggle hosts a number of data science competitions featuring real-world data sets and hefty cash prizes. You’ll also often be tasked with solving real-world problems like: how to classify toxic online comments, forecast restaurant visitors, or recognize objects from space.
          </p>
         </li>
        </ol>
        <h2 id="share-your-side-project">
         <a aria-hidden="true" class="header-anchor" href="#share-your-side-project">
          <i aria-hidden="true" class="fa fa-link">
          </i>
         </a>
         <strong>
          Share Your Side Project
         </strong>
        </h2>
        <p>
         One of the toughest things about working on a side project is getting anybody other than you, your spouse, and your pet cat Millie to appreciate it. Please use the comments on this article as an opportunity to get some love for your latest side project. Share a link to what you’re working on, and let us know:
        </p>
        <ul>
         <li>
          What has been the biggest challenge while working on your side project?
         </li>
         <li>
          What’s one thing you’ve learned working on your side project?
         </li>
         <li>
          What tips would you give to someone else starting a similar side project?
         </li>
        </ul>

For example, we may unintentionally limit the types of projects that we feel are ‘worthy’ as side projects.\n\nBy setting the bar too high — like aiming to create a successful, popular, and original open source library — we rob ourselves of the opportunity to have fun and learn something from a less ambitious, but still worthwhile, side project.\n\nIn this post, I want to help fight “side project paralysis” by offering up 40 side project ideas that any software developer can start in one weekend. The emphasis here is not on projects that are going to change the world. Instead, these are fun projects to give you something to add to your portfolio **and** make you a better developer. \n\n## **Computer Science Side Projects**\n\n*These projects are suited for engineers trying to beef up their computer science fundamentals. They’ll help you learn more about specific aspects of computing, like file transfer and operating systems design. These projects can be particularly useful for self-taught developers who’ve focused more on practical skills than comp sci fundamentals.*\n\n1. **FTP Client** (suggested implementation: web, desktop or CLI app). \nBuild a simple FTP client. As a bonus challenge, support secure file transfer.\n\n2. **HTTP Server** (suggested implementation: any programming language you want to master). To understand HTTP deeply, build an HTTP server. There are a number of [tutorials](https://ruslanspivak.com/lsbaws-part1/) available to guide you through the process.\n\n3. **Build a simple operating system** (suggested implementation: any programming language you want to master). This is a project that is technically challenging and will deepen your understanding of how computers and operating systems work. You might wish to start with the free (and cleverly named) book, [Operating Systems: From 0 to 1](https://tuhdo.github.io/os01/).\n\n4. **Build a web scraper** (suggested implementation: any programming language you want to master). Build a tool that takes a URL as input and returns the content of the URL as HTML or XML.\n\n5. **Bandwidth monitor** (suggested implementation: live updating CLI tool). Build a tool to track how much data you have downloaded or uploaded on the internet. Have it email you a weekly report of your usage. As a bonus challenge, predict peak usage times.\n\n6. **Implement a cipher** (suggested implementation: any programming language you want to master). Build a tool that takes a string of text as input and encrypts it using a cipher, such as the [Caesar cipher](https://en.wikipedia.org/wiki/Caesar_cipher). \n\n7. **Create a text message downtime alerter** (suggested implementation: app hooked up to the [Twilio API](https://www.twilio.com/docs/api/rest) for sending text messages). Create an app that takes a URL as input and sends you a text message when a request to the URL returns a response code other than 200 (OK).\n\n## **Personal Finance Side Projects**\n\n*These projects will help you achieve a practical goal (get a better handle on your finances), while also improving your software engineering skills.*\n\n8. **A net worth calculator and tracker** (suggested implementation: CLI, web, or mobile app). Build a calculator you can use to track the rise or fall of your net worth on a monthly basis. You can use something like this [net worth worksheet](http://www.schwabmoneywise.com/public/file/P-4038856/Net-Worth-Worksheet.pdf) from Charles Schwab to guide you. Optional extension: have it send you a 12 month report for the previous year on the first of January each year.\n\n9. **A tax forecaster** (suggested implementation: web app). This will be particularly useful if you do any freelance software engineering. Build a tool that takes your freelance earnings as input and then predicts your expected tax liability for the rest of the financial year. Make it smart enough to predict periods of higher or lower demand for your services, and adjust accordingly.\n\n10. **A deal finder** (suggested implementation: web app with mobile notifications). Build a simple web app to notify you when an item you covet goes on sale for a good price. You could use a web scraper to pull the item’s product page and notify you of any price changes.\n\n11. **An expense tracker** (suggested implementation: web or mobile app). Create a simple interface you can use to add and categorize your expenses. Generate monthly reports based on the inputs and write custom alerts for things, like, “spending too much money on coffee… as always.”\n\n12. **A financial independence calculator** (suggested implementation: web app). [Financial independence](https://www.reddit.com/r/financialindependence/) is, essentially, saving and investing as much of your income as possible so that you don’t *need* to work for money. While many financially independent people continue to work, they can now focus on doing work that they love, rather than work that pays the most. Build a tool to calculate, based on your: current savings, investments, income, retirement accounts, and expenses, how far away you are from financial independence. Some examples: [FIREcalc](https://www.firecalc.com/), [cFIREsim](http://www.cfiresim.com/).\n\n13. **A bill splitter** (suggested implementation: mobile app). Build a simple tool to help you and your friends split bills when you go out to eat together.\n\n## **Games and Simulation Side Projects**\n\n*Most software engineers I know are fascinated by the world of game development, graphics, and simulations, but don’t have a lot of experience with them. These projects are small enough that you can set foot into this world without biting off more than you can chew.*\n\n14. **A random name generator** (suggested implementation: CLI, web, or mobile app). Build a random name generator ([example](https://www.behindthename.com/random/)) that creates unique names on the fly, based on an algorithm. Use machine learning techniques to help you by training the program with a sample data set of names similar to those you want to generate. Otherwise, create your own lexical rules for how names are generated. For example, a name generating algorithm inspired by [The Handmaid's Tale](https://en.wikipedia.org/wiki/The_Handmaid%27s_Tale) might stipulate that names for Handmaid women should start with 'Of', and end with a random male name, e.g., Ofpeter.\n\n15. **[Conway’s Game of Life](https://en.wikipedia.org/wiki/Conway%27s_Game_of_Life)** (suggested implementation: any platform capable of real-time graphical rendering). Conway’s Game of Life simulates the lives of simple cells that obey algorithmic laws. This video explains how the game works and includes an example of one possible result:\n\n@[epic conway's game of life](https://www.youtube.com/watch?v=C2vgICfQawE\u0026feature=player_embedded)\n\n16. **A procedurally generated map maker** (suggested implementation: browser-based app). Create a browser-based application that allows users to procedurally generate a terrain map based on a random seed. The map can be as detailed or as simple as you’d like. This project is a good opportunity to learn about [procedural generation](https://en.wikipedia.org/wiki/Procedural_generation).\n\n17. **A character generator** (suggested implementation: browser-based or mobile app, CLI). Create a tool that allows you to randomly generate playable characters for your favorite role-playing games, whether they be tabletop games, like Pathfinder, or video games, like Divinity: Original Sin.\n\n18. **Interactive fiction** (suggested implementation: CLI). A fun way to get into game development without having to worry about graphical assets, interactive fiction renders the world for the player through text descriptions. [The Dreamhold](http://eblong.com/zarf/zweb/dreamhold/) is a good example of interactive fiction with a useful ‘help’ command.\n\n## **Artificial Intelligence Side Projects**\n\n*AI’s usefulness in day-to-day software engineering is increasing by leaps and bounds. It’s now easier than ever before to make your first forays into the world of Artificial Intelligence.*\n\n19. **An unbeatable Tic-Tac-Toe engine** (suggested implementation: CLI program). For an excellent first AI project, try to write an engine that cannot be beaten at Tic-Tac-Toe. You can achieve this by implementing this [strategy](https://en.wikipedia.org/wiki/Tic-tac-toe#Strategy), which produces a draw as its worst-case outcome.\n\n20. **A chess engine** (suggested implementation: an engine written in a programming language you want to learn, or master). Try to write an engine that can play chess against a human opponent using a [Universal Chess Interface](http://wbec-ridderkerk.nl/html/UCIProtocol.html) compatible GUI, such as XBoard. See [Stockfish](https://github.com/official-stockfish/Stockfish) as an example. For a less daunting challenge, you may wish to focus on the behavior of just one piece, e.g., Knights.\n\n21. **A niche chatbot** (suggested implementation: web app). Some of the greatest minds in the world are working on chatbots that respond in lifelike ways. It’s an incredibly difficult challenge, but, by reducing the scope of your chatbot, you have a side project that is more approachable for evenings and weekends. Create a chatbot that produces real-sounding responses based on a niche topic that you’re passionate about: your favorite band, video game, sports team, or TV show. Leverage an existing library to help you, such as [ChatterBot](https://github.com/gunthercox/ChatterBot).\n\n22. **A spam classifier** (suggested implementation: any programming language you want to master). Build a tool to classify whether an email is or isn’t spam based on the content alone. You can use this [public data set of emails from the Enron investigation](https://www.kaggle.com/wcukierski/enron-email-dataset) to test your spam classifier.\n\n## **Entertainment Side Projects**\n\n*These projects are for software engineers who want to work on something fun and light-hearted.*\n\n23. **A movie showtime finder** (suggested implementation: web or mobile app with email or text message notifications). Build a program that notifies you, by text or email, about showtimes for potentially interesting movies playing at your favorite cinema. The concept of an 'interesting movie' can be derived using machine learning (if you watch enough movies to have good training data), or a handcrafted algorithm. For example, you might use the [Open Movie Database API](http://www.omdbapi.com/), paired with an HTML parser like [BeautifulSoup](https://www.crummy.com/software/BeautifulSoup/bs4/doc/), to build a program that alerts you to sci-fi movies rated 7.0 or above on IMDB, movies starring Amy Adams, and/or any movie with an average rating of 8.0 or higher.\n\n24. **A spoiler blocker** (suggested implementation: browser extension). Short of implementing a total social media/internet/watercooler chat ban, it can be difficult to avoid seeing spoilers for your favorite TV show… especially if that TV show is as popular as Game of Thrones. Create a browser extension that removes all mentions of your favorite show from any webpage loaded in your browser… or replaces them with cute pictures of puppies.\n\n## **Fun Side Projects**\n\n*These projects are all different, from logging sensor data to finding new desktop backgrounds for your computer. One thing they all have in common is that they’re fun, and several of them will have you work with interesting APIs.*\n\n25. **Pixel art generator** (suggested implementation: any programming language you want to master). Build a tool that takes an image as input and samples the image to produce pixel art as output. If you want to improve your front-end skills, generate the resulting pixel art using CSS.\n\n26. **Music suggestion tool** (suggested implementation: build a wrapper for the [Spotify API](https://developer.spotify.com/web-api/get-the-users-currently-playing-track/)). Create a tool that tracks the music you listen to and generates a playlist with similar qualities, but of songs you haven’t heard before. The Spotify API provides all of the tools needed to extract what you’ve listened to and create a playlist — the recommendation engine is up to you!\n\n27. **Temperature logger** (suggested implementation: Raspberry Pi, temperature sensor, web app). Have your Raspberry Pi hooked up to a temperature sensor and send temperature data from your home to a web app that saves the data to a database or updates a CSV file. For bonus points, have a weekly temperature report emailed to you. Other Raspberry Pi projects: [home automation](https://electronicsofthings.com/videos/home-automation-hand-gesture/), [home security](https://electronicsforu.com/electronics-projects/iot-video-surveillance-raspberry-pi).\n\n28. **Microlearning app** (suggested implementation: web or mobile app). Build an application that sends you one page per day about something you want to learn. This could be a random page from Wikipedia, a page of [React](https://reactjs.org/docs/hello-world.html) documentation, a [Kanji](https://jisho.org/) character, or a page from the [CIA World Factbook](https://www.cia.gov/library/publications/the-world-factbook/).\n\n29. **Slack bot** (suggested implementation: [Slack API](https://api.slack.com/)). If you or your team use the popular chat app Slack, build a bot to make some aspect of your (or your team’s) life easier. Ideas: a coffee order bot, a bot that reports daily on the number of commits made to your team repos, or a daily stand-up reminder bot.\n\n30. **Daily desktop background** (suggested implementation: [Unsplash API](https://unsplash.com/developers), scripting language for your OS). Build an app that refreshes your desktop background with a new image every day.\n\n## **Coding Challenge Side Projects**\n\n*Rather than building a project from scratch, these are collections of coding challenges to improve your skills. If you want to get better at solving discrete problems, or improving your chops with a new programming language, these are all good options.*\n\n31. **[Advent of Code](http://adventofcode.com/)** (suggested implementation: any language you want to practice). Advent of Code challenges occur every year in the days leading up to Christmas. They include lots of fun programming challenges that can help you develop your chops in a programming language you’re learning or one you want to master. The challenges from [2015](https://adventofcode.com/2015), [2016](https://adventofcode.com/2016), and [2017](https://adventofcode.com/2017) are still available.\n\n32. **[Kaggle’s Titanic Challenge](https://www.kaggle.com/c/titanic)** (suggested implementation: Python or R). Kaggle offers a fascinating challenge to introduce you to the basics of machine learning with Python or R: use a real data set from the Titanic passenger log to predict which passengers were most likely to survive the disaster.\n\n33. **[Project Euler](https://projecteuler.net/archives)** (suggested implementation: any programming language you want to practice). Project Euler challenges are dedicated to exploring the marriage between mathematics and programming. Use programming tools to solve various mathematical challenges.\n\n## **Daily Life Side Projects**\n\n*These side projects have a practical purpose: they help with some aspect of your daily life. Though they solve simple real-world problems (things like what to have for lunch), the solutions can be as simple, or as complex, as you’d like.*\n\n34. **Lunch picker** (suggested implementation: CLI, web or mobile app). If you’re a working software engineer, you’ve probably wrestled with one of the toughest questions in software development… where should I have lunch? Your lunch picker is the tool you’ll turn to to answer this question. It can be super simple and pick from a range of options you know you like at random, or more complex — pulling in data from Google Reviews and taking into account: location, price, and type of cuisine.\n\n35. **Date planner** (suggested implementation: mobile app). One of the hardest things about dating, or being in a long-term relationship, is deciding where to go on dates. Build a tool that scours restaurant reviews, event calendars, and other data for date idea suggestions.\n\n36. **Tool to simplify your email newsletters** (suggested implementation: scripts hooked up to a mail server). If you’re like me, you receive way too many email newsletters every week and they clutter up your inbox. Instead of unsubscribing from all of them in a fit of rage, build a tool that captures these emails and compiles them all into a single email that is sent once a week.\n\n37. **‘Bring your umbrella’ notifier** (suggested implementation: mobile app). Build a simple mobile app that sends you a phone notification in the morning telling you if you should bring your umbrella to work (e.g., if it is going to rain in your area that day).\n\n38. **Collection tracker** (suggested implementation: web or mobile app). Build a tool to keep track of something you collect. You could keep track of: the value of the item, the year it was made, its condition, and its location.\n\n## **Data Side Projects**\n\n*These projects will improve your ability to work with data. Software engineers work with ever-larger amounts of data. These projects will help you learn how to collate, parse, and analyze data with confidence.*\n\n39. **Create and automatically update a data set** (suggested implementation: web scraper input to CSV output). Build a tool to automatically build and update a data set about something you’re interested in. Suggested data sets: stats about your favorite sports team, flights to destinations you want to visit, meteorological data from where you live, or anything else that interests you. The most important thing is that the data set should require constant updates as new data is generated — and should occur automatically. For example, as soon as your sport’s team’s results are posted on a website, the data should be automatically scraped and added to your data set.\n\n40. **Enter a data science competition** (suggested implementation: R or Python). Kaggle hosts a number of data science competitions featuring real-world data sets and hefty cash prizes. You’ll also often be tasked with solving real-world problems like: how to classify toxic online comments, forecast restaurant visitors, or recognize objects from space.\n\n## **Share Your Side Project**\n\nOne of the toughest things about working on a side project is getting anybody other than you, your spouse, and your pet cat Millie to appreciate it.
